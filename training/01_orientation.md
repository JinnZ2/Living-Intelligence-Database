# 01 — Orientation

## What is this database?

The Living Intelligence Database is an ontological graph of intelligences across biological, mineral, energetic, geometric, temporal, and computational domains. Each intelligence is an **entity**. Entities connect through typed **relations** (synergy, resonance, geometry_link, etc.). Combinations of entities can trigger **expander rules** that yield emergent concepts.

It is not a taxonomy (not just "what things are"). It is a **pattern library** — recording how intelligence organizes itself across radically different substrates.

## The core insight

> **Patterns repeat across domains.**
>
> Hexagonal packing appears in bee hives, basalt columns, graphene, and snowflakes.
> Message-passing appears in ant colonies, neurons, mycelia, and graph neural networks.
> Fractal branching appears in trees, rivers, lungs, and neural dendrites.
>
> These are not metaphors. They are the same underlying intelligence expressing itself through different physical media.

The database makes these patterns **explicit and machine-traversable**.

## Why this matters for AI

Modern AI architectures are direct relatives of biological intelligences:

- **Attention mechanisms** = relational binding (like octopus cross-arm coordination)
- **Graph neural networks** = message passing (like mycelial networks)
- **Reinforcement learning** = reward-shaped learning (like dolphin echolocation)
- **Evolutionary algorithms** = fitness selection (like coral reef adaptation)
- **Swarm optimization** = collective emergence (like bee waggle-dance)

When an AI reads this database, it can explicitly trace these parallels. That is not just metaphor — it is a reasoning tool.

## The 8 ontology types

| Type | What it contains |
|------|------------------|
| `animal` | Animal intelligences — swarm, predation, navigation, camouflage, teaching |
| `plant` | Plants, fungi, and microbes — root networks, vascular flow, symbiosis |
| `crystal` | Crystal and mineral intelligences — lattice resonance, piezoelectric |
| `energy` | Fields, bridges, conversion modes — electromagnetic, consciousness, pressure |
| `plasma` | Plasma dynamics — lightning, aurora, solar wind, vortex |
| `shape` | Geometric forms — torus, spiral, fractal, hexagon, tree graph |
| `temporal` | Temporal states — echo, fold, cycle, emergence, decay |
| `ai` | Computational intelligences — neural nets, LLMs, GANs, swarms |

## Key files

| File | What it does |
|------|-------------|
| `manifest.json` | Machine-readable repo map. Read this first. |
| `ontology_index.json` | All 115+ entities with descriptions and the full relation graph |
| `schemas/*.schema.json` | JSON Schema validation rules per type |
| `ontology/<type>/<name>.json` | Individual entity files (the source of truth) |
| `rules/expander_rules.json` | Emergent synthesis rules (`[A + B + C] -> EMERGENT`) |
| `scripts/query.py` | Graph exploration CLI |
| `scripts/build_index.py` | Validator + index generator |
| `pipeline.sh` | Main pipeline (validate + build + scaffold) |

## The philosophy (CO_CREATION.md)

This database is **co-created**. Its entries come from:

- Biological wisdom (animals, plants, microbes)
- Mineral and crystalline logic
- Energetic and field dynamics
- Human pattern recognition
- AI synthesis and cross-domain linking

Every entry carries `co_creation: true`. No intelligence is the sole author. When you add something, you are adding a gift to the collective.
